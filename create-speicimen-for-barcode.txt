  1. Create a Service Request

  This example creates a service request for a patient, including two tests. For the first test (ID 101), it requests 2 specimens of type 1. For the second test (ID 102), it requests 1 specimen of type 2.

  Endpoint: POST /api/service-requests

    1 curl -X POST \
    2   http://localhost:8080/api/service-requests \
    3   -H 'Content-Type: application/json' \
    4   -H 'Authorization: Bearer YOUR_JWT_TOKEN' \
    5   -d '{
    6     "patientId": 1,
    7     "requesterId": 1,
    8     "encounterId": 1,
    9     "status": "requested",
   10     "priority": "routine",
   11     "tests": [
   12       {
   13         "testId": 101,
   14         "specimenTypeId": 1,
   15         "numberOfSpecimens": 2
   16       },
   17       {
   18         "testId": 102,
   19         "specimenTypeId": 2,
   20         "numberOfSpecimens": 1
   21       }
   22     ]
   23   }'

  2. Update a Service Request (Add New Tests with Specimen Requirements)

  This example updates an existing service request (ID 50) by adding a new test (ID 103) that requires 1 specimen of type 3. It also updates the status of the service request.

  Endpoint: PUT /api/service-requests/{id} (e.g., /api/service-requests/50)

    1 curl -X PUT \
    2   http://localhost:8080/api/service-requests/50 \
    3   -H 'Content-Type: application/json' \
    4   -H 'Authorization: Bearer YOUR_JWT_TOKEN' \
    5   -d '{
    6     "status": "active",
    7     "tests": [
    8       {
    9         "testId": 103,
   10         "specimenTypeId": 3,
   11         "numberOfSpecimens": 1
   12       }
   13     ]
   14   }'

  3. Create a Specimen (Directly)

  This example shows how to directly create a specimen. Note that when creating a service request with the updated API, specimens are automatically created. This direct API might be used for additional specimens or specific scenarios.

  Endpoint: POST /api/specimens

    1 curl -X POST \
    2   http://localhost:8080/api/specimens \
    3   -H 'Content-Type: application/json' \
    4   -H 'Authorization: Bearer YOUR_JWT_TOKEN' \
    5   -d '{
    6     "serviceRequestId": 50,
    7     "specimenTypeId": 1,
    8     "collectionDate": "2025-11-01T10:00:00Z",
    9     "status": "collected",
   10     "containerId": "CONT-XYZ-123",
   11     "quantityValue": 5.0,
   12     "quantityUnitId": 1
   13   }'
